## 产品概述

完善 KnowDo 企业级 AI 知识平台前端所有页面的未完成交互功能，包括模型管理页面的添加/配置功能、收藏夹页面的筛选与移动功能、收藏状态与收藏夹的同步、以及添加 404 页面等。

## 核心功能

- **模型管理页面**：实现"添加模型"弹窗表单（支持选择模型类型、填写名称、API地址、密钥、参数等）、"配置"按钮打开配置抽屉（可编辑模型参数）、完善"刷新状态"按钮的模拟刷新逻辑
- **收藏夹页面**：修复按收藏夹筛选逻辑（当前切换收藏夹但右侧仍显示所有收藏知识）、实现"移动至收藏夹"功能、修复 toggleFavorite 与 favoriteFolders.knowledgeIds 的同步问题
- **404 页面**：创建 NotFound 组件并添加到路由通配符
- **AI 工具栏**：为知识创建页面的 AI 辅助按钮实现模拟交互效果（优化排版、生成摘要、智能翻译、智能校对）

## 技术栈

- 前端框架：React 19 + TypeScript
- UI 组件库：Ant Design 6
- 状态管理：Zustand
- 样式方案：Tailwind CSS 4 + CSS 变量
- 构建工具：Vite 8

## 实现方案

### 1. 模型管理页面（ModelConfig）

**添加模型功能**

- 使用 Ant Design `Modal` + `Form` 创建添加模型弹窗
- 表单字段：模型类型（LLM/Embedding/Reranker）、模型名称、模型标识、供应商、API地址、API密钥、最大Token、并发数、超时时间
- 提交后将新模型添加到对应类型的 `modelList` 中
- 在 store 中新增 `addModel` action

**配置按钮功能**

- 使用 Ant Design `Drawer` 创建配置面板
- 可编辑：API地址、API密钥、最大Token、并发数、超时时间、温度参数（LLM）
- 支持测试连接、保存配置
- 在 store 中新增 `updateModel` action

**刷新状态功能**

- 模拟刷新：遍历当前筛选后的模型，依次调用测试连接，更新 `lastTest` 和 `testResult`

### 2. 收藏夹页面（Favorites）

**修复收藏夹筛选逻辑**

- 当前 `currentFolder` 已获取但实际显示时未按 `knowledgeIds` 过滤
- 修改右侧知识列表，根据 `currentFolder.knowledgeIds` 过滤显示的收藏知识

**实现移动至收藏夹功能**

- 在知识卡片上添加"移动"按钮或右键菜单
- 点击后弹出选择收藏夹的 `Select` 或 `Modal`
- 调用 store 中的 `moveToFolder` 方法

**修复 toggleFavorite 同步问题**

- 当前 `toggleFavorite` 只更新 `knowledgeList` 中的 `isFavorited` 和 `favoriteCount`
- 需要同步更新 `favoriteFolders` 中默认收藏夹的 `knowledgeIds`
- 修改 store 中的 `toggleFavorite` action

### 3. 404 页面

- 创建 `src/pages/NotFound/index.tsx` 组件
- 使用 Ant Design `Result` 组件展示 404 状态
- 添加返回首页的链接按钮
- 在 `App.tsx` 路由中添加 `<Route path="*" element={<NotFound />} />`

### 4. AI 工具栏模拟交互

- "AI 优化排版"：模拟加载 1.5s 后提示"排版已优化"
- "生成摘要"：从正文提取前 200 字作为摘要，填入摘要输入框
- "智能翻译"：模拟加载 2s 后提示"翻译完成（模拟）"
- "智能校对"：模拟加载 1s 后显示找到的问题列表（模拟数据）

## 关键代码结构

store 中需要新增的 action：

```typescript
// 新增模型
addModel: (model: Omit<ModelConfig, 'id' | 'status' | 'lastTest' | 'testResult'>) => void;

// 更新模型配置
updateModel: (id: string, data: Partial<ModelConfig>) => void;
```

## 页面设计

### 1. 添加模型弹窗（Modal）

- 弹窗标题："添加模型"，宽度 600px
- 表单布局：垂直布局，Label 在上方
- 字段顺序：模型类型（Radio.Group）、模型名称（Input）、模型标识（Input）、供应商（Select）、API地址（Input）、API密钥（Input.Password）、最大Token（InputNumber）、并发数（InputNumber）、超时时间（InputNumber）
- 底部：取消 + 确认按钮

### 2. 模型配置抽屉（Drawer）

- 抽屉标题："模型配置 - {模型名称}"，宽度 480px
- 表单字段：API地址、API密钥、最大Token、并发数、超时时间、温度（Slider，仅LLM类型显示）
- 底部：测试连接按钮 + 保存按钮

### 3. 收藏夹移动弹窗

- 使用 Modal 或 Popover 展示收藏夹列表
- 点击收藏夹名称即执行移动操作
- 显示"取消收藏"选项

### 4. 404 页面

- 使用 Ant Design Result 组件
- 状态：404，标题："页面未找到"，副标题："抱歉，您访问的页面不存在"
- 操作按钮："返回首页"（链接至 `/`）

## 交互细节

- 添加模型成功后自动刷新对应 Tab 的模型列表
- 配置抽屉打开时自动填充当前模型数据
- 测试连接显示 loading 状态，完成后显示成功/失败提示
- 移动至收藏夹操作后显示成功提示
