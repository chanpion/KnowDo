# 知识库详情页 UI 优化设计

## 概述

对知识库详情页（`/detail/:id`）进行 UI/UX 优化，保持现有两栏布局结构，优化头部信息展示、文档卡片样式、设置面板中分段策略配置、召回测试界面精简。

## 当前问题

1. 头部信息松散，名称/类型/状态/描述排列缺乏层次
2. 文档卡片缺少操作菜单和悬停预览
3. 设置面板缺少分段策略配置
4. 召回测试查询区和结果展示布局可以更紧凑

## 设计方案

### 1. 页面布局（不变）

保持左侧菜单 200px + 右侧内容 flex-1 的两栏布局，侧边栏菜单项：文档、召回测试、设置。

### 2. 头部区域

**结构：**

```
[← 返回] [知识库名称] [类型标签] [状态徽标]                    [资源授权] [关联资源]
[描述文字（单行省略）]
[文档数 8] [分段数 156] [字符数 128K] [更新于 2026-06-28]
```

- 名称 + 类型标签 + 状态徽标 在同一行
- 操作按钮右对齐（资源授权、关联资源）
- 描述文字单行省略展示
- 底部添加统计条：文档数、分段数、字符数、更新时间

**样式：**
- 根据知识库类型（通用/Web/飞书）使用不同渐变背景色
- 使用 `.detail-header-gradient` 类扩展
- 调整 padding 和内边距

### 3. 文档卡片

**卡片内容：**
- 头部：类型徽标（文档/图片/链接等）+ 标题 + 右上角 ⋮ 操作菜单
- 中间：摘要文字（最多两行省略）
- 标签行：最多 3 个标签
- 底部：作者·部门 + 发布时间 + 阅读量

**操作菜单（Dropdown）：**
- 编辑 → 导航到 `/create?edit=:id`
- 归档 → 调用 `archiveKnowledge`
- 复制链接 → 复制当前文章链接
- 删除 → Popconfirm 确认后删除

**交互：**
- hover 时轻微上浮（+2px translateY + 阴影增强）
- 右上角 ⋮ 按钮 hover 时显示背景色

### 4. 分段策略配置

在设置面板新增「分段策略」配置区域：

**模式选择：**
- 智能分段（默认）- 自动优化分段参数，仅显示最大字符数和重叠字符数滑块
- 高级分段 - 可配置分段符号、分段长度、重叠长度
- QA 问答对 - 用于 Q&A 模式，显示问题+答案对的编辑界面

**预览区域：**
- 显示当前策略下文档的分段效果预览（前 3 段）
- 每段显示序号和内容摘录

**涉及修改：**
- `src/store/index.ts` — 在 `Dataset` 类型中 `chunkStrategy` 已存在
- 更新 `updateDataset` action 以支持 `chunkStrategy` 字段修改

### 5. 召回测试界面精简

**查询区：**
- 输入框 + 测试按钮 在一行紧凑排列
- TopK 选择器和检索模式选择器整合到参数行
- 参数行使用浅灰背景区分

**结果展示：**
- 每条结果：文档名称 + 分段位置 + 匹配分数（颜色编码高/中/低）
- 内容展示：左侧颜色指示条 + 内容摘要 + 关键词高亮
- 空状态：显示引导提示文字

## 不做的事项

- 不修改路由结构
- 不改动数据模型
- 不实现真实后端接口（保持 mock 模式）
- 不实现文章详情页（`/article/:id`）
- 不实现召回测试的真实向量检索

## 验证方式

1. 启动 dev server: `cd knowdo-frontend && npm run dev`
2. 访问 `/detail/1` 验证头部展示
3. 切换侧边栏菜单验证各面板
4. 验证文档卡片 hover 效果和操作菜单
5. 验证设置面板中分段策略配置
6. 验证召回测试查询和结果展示
7. 运行 `npm run lint` 确保无语法问题
